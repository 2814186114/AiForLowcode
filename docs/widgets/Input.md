# Input 输入框

通过鼠标或键盘输入内容，是最基础的表单域的包装。

## 何时使用

- 需要用户输入表单数据时。
- 提供用户可编辑的文本输入区域。
- 需要对用户输入内容进行即时反馈时。

## 代码演示

### 基本用法

最基本的输入框用法。

```jsx
<Input label="输入框" />
```

### 带占位符

设置输入框的占位符文本。

```jsx
<Input 
  label="带占位符" 
  placeholder="请输入内容..." 
/>
```

### 预设值

设置输入框的默认值。

```jsx
<Input 
  label="预设值" 
  value="这是预设值" 
/>
```

### 表单中使用

在表单中使用输入框，配合onChange事件处理输入内容。

```jsx
<Input 
  label="用户名" 
  placeholder="请输入用户名" 
  onChange={(value) => console.log('输入值:', value)} 
/>
```

## API

| 属性 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| label | 输入框标签文本 | string | `'输入框'` |
| placeholder | 输入框占位符文本 | string | `'请输入'` |
| value | 输入框的值 | string | `''` |
| onChange | 输入内容变化时的回调函数 | function | `() => { }` |
| id | 组件唯一标识，用于设计器内部状态管理 | string | - |

### 事件

| 事件名 | 说明 | 回调参数 |
| --- | --- | --- |
| onChange | 输入内容变化时触发 | `(value: string) => void` |

## 注意事项

1. 输入框在设计模式下（mode='design'）是只读的，仅可拖拽。
2. 在预览模式下（mode='preview'）可以正常输入和编辑。
3. 当输入框的值发生变化时，会触发onChange事件，并将当前值作为参数传递。
4. 如果设置了id属性，输入框会监听状态管理中的属性变化，自动更新组件状态。
